# /commit スキル

プロジェクトのコミット規約に沿って変更をコミットする。

## 手順

### 1. 変更内容の把握

```bash
git status
git diff --cached --stat
git diff --stat
```

ステージされた変更と未ステージの変更を確認する。

### 2. 品質チェック（コミット前）

```bash
npm run check:smart
```

エラーがある場合はユーザーに報告し、先に `/fix` の実行を提案する。

### 3. 変更のステージング

未ステージの変更がある場合、関連するファイルをステージする。

- 機密ファイル（`.env`、認証情報等）は除外する
- `build-output/` はステージしない（自動生成のため）
- `types/generated-*/` はステージしない（自動生成のため）

### 4. コミットメッセージの作成

以下のフォーマットに従う:

```
<type>: 変更内容の要約（日本語）

詳細な説明（必要に応じて）

🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

#### type の選択基準

| type       | 用途                             |
| ---------- | -------------------------------- |
| `feat`     | 新機能の追加                     |
| `fix`      | バグ修正                         |
| `refactor` | 振る舞いを変えないコード改善     |
| `style`    | フォーマット・コメントのみの変更 |
| `docs`     | ドキュメントの変更               |
| `test`     | テストの追加・修正               |
| `chore`    | ビルド・設定・ツールの変更       |

#### ルール

- **要約行は日本語で書く**
- 要約行は簡潔に（50文字目安）
- 詳細説明は「なぜ」を中心に書く
- 署名行（`🤖 Generated with ...`）を必ず含める

### 5. コミットの実行

```bash
git commit -m "<message>"
```

### 6. 結果の確認

```bash
git log --oneline -1
```

コミットが正しく作成されたことを確認し、ユーザーに報告する。
