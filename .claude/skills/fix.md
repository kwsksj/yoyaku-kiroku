# /fix スキル

コードのフォーマット、Lint、型エラーを自動修正し、品質チェックが通る状態にする。

## 手順

### 1. 現状の確認

まず品質チェックを実行し、エラーの有無と種類を把握する。

```bash
npm run check:smart 2>&1
```

エラーがなければ「問題ありません」と報告して終了。

### 2. 自動修正の実行

自動修正可能なエラー（フォーマット、Lint）を一括修正する。

```bash
npm run validate:fix
```

### 3. 自動修正できないエラーの対応

`validate:fix` で解消しないエラーがある場合、手動で修正する。

#### 型エラーの場合

- エラーメッセージとファイル:行番号を確認
- JSDoc の `@param` / `@returns` 型を修正
- `Object` / `any` → 具体的な Core 型に変更
- オプショナルアクセス（`?.` / `??`）の不足を補う
- 型を変更した場合は `npm run types:refresh` を実行

#### Lint エラーの場合

- 未使用変数の削除
- import 制約違反の修正（禁止パターンを許可パターンに書き換え）
- その他の ESLint ルール違反の修正

### 4. 修正結果の検証

```bash
npm run check:smart
```

すべてのチェックが通ることを確認する。エラーが残っている場合は手順 3 に戻る。

### 5. 結果報告

```markdown
## 修正結果

- フォーマット: N 件修正
- Lint: N 件修正
- 型エラー: N 件修正
- 最終チェック: ✅ 全パス
```
